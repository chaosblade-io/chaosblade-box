#### 接口说明：任务执行历史（分页）
+ 方法与路径: GET /api/detection-tasks/{taskId}/executions
+ 作用: 按开始时间倒序返回指定任务的执行历史，支持分页

#### 返回数据结构
+ 最外层统一包装: ApiResponse<PageResponse>
    - success: 布尔，是否成功
    - error: 失败时的错误信息对象（包含 errorCode、message、details），成功为 null
    - data: PageResponse，分页内容
+ PageResponse 字段
    - items: TaskExecution 列表，按 startedAt 降序排列
    - total: long，总记录数
    - page: int，当前页码（从 1 开始）
    - size: int，每页大小
+ TaskExecution 字段（每条执行记录）
    - id: 执行记录 ID（bigint，自增）
    - status: 执行状态（字符串枚举）
        * INIT
        * GENERATING_CASES
        * ANALYZING_PATTERNS
        * RECORDING_READY
        * INJECTING_AND_REPLAYING
        * RULES_READY
        * LOAD_TEST_BASELINE
        * DONE
        * FAILED
    - startedAt: 开始时间（yyyy-MM-dd HH:mm:ss）
    - finishedAt: 结束时间（yyyy-MM-dd HH:mm:ss，可为空）
    - durationSeconds: 执行时长（Long，计算字段）
        * 若 finishedAt 不为空：finishedAt - startedAt（秒）
        * 若未结束：当前时间 - startedAt（秒）
    - namespace: 命名空间（如业务系统标识）
    - requestNum: 并发请求数（int）
    - errorCode: 错误码（可为空）
    - errorMsg: 错误信息文本（可为空）

#### 排序与分页
+ 排序: startedAt 倒序（最新的执行记录排在前面）
+ 分页参数:
    - page: 页码，默认 1，取值需 >= 1
    - size: 每页大小，默认 20，取值需 >= 1

#### 备注
+ durationSeconds 为服务端基于 startedAt/finishedAt 动态计算的只读字段，无需在数据库中存储
+ status 为字符串枚举，请前端按上述枚举值进行展示与状态流转映射
+ 如需同时返回“是否有正在执行中的实例”的布尔标志，可在列表外增加聚合标志（基于运行中状态集合判断），需要的话我可以在接口中扩展 data 结构添加 running 字段

#### 接口说明：获取检测任务详情（增强版，聚合）
+ 方法与路径: GET /api/detection-tasks/{taskId}
+ 作用: 返回指定任务的“全量聚合详情”，便于前端一次性渲染任务、系统、拓扑、故障与 SLO 配置以及最新执行状态

#### 返回数据结构
+ 最外层统一包装: ApiResponse
    - success: 布尔，是否成功
    - error: 失败时的错误对象（包含 errorCode、message、details），成功为 null
    - data: DetectionTaskDetails 聚合对象
+ DetectionTaskDetails 字段
    - task: DetectionTask
        * 任务基础信息（id, name, description, systemId, apiId, createdBy, updatedBy, requestNum, createdAt, updatedAt, archivedAt 等，具体以实体为准）
    - sys: System
        * 通过 task.systemId 查询到的系统详情（如 id, name, owner, createdAt 等，以实体为准）
    - apiDefinition: HttpReqDef
        * 任务关联的 HTTP 请求定义详情（若模型暂未直接关联请求定义，则可能返回 null；后续若补充关联，将返回对应的 HttpReqDef）
    - topology: ApiTopology
        * 指定 systemId 与 apiId 的拓扑主对象（以实体为准）
    - topologyNodes: List
        * 上述 topology 的节点列表（节点字段以实体为准，例如 id、name、serviceName 等）
    - topologyEdges: List
        * 上述 topology 的连边列表（边字段以实体为准，例如 id、fromNodeId、toNodeId、protocol 等）
    - faultConfigs: List
        * 该任务 task_id 下的所有故障配置（字段以实体为准：id, nodeId, faultscript(JSON字符串), type, taskId）
    - taskSlos: List
        * 该任务 task_id 下的所有 SLO 配置（字段以实体为准：id, p95, p99, errRate, taskId, nodeId）
    - latestExecutionStatus: String
        * 最新一次 task_execution 记录的状态（按 started_at 降序取第一条）
        * 可能值：INIT, GENERATING_CASES, ANALYZING_PATTERNS, RECORDING_READY, INJECTING_AND_REPLAYING, RULES_READY, LOAD_TEST_BASELINE, DONE, FAILED；若无记录为 null

参考 DTO 定义（节选）：

```plain
public static class DetectionTaskDetails {
  public DetectionTask task;
  public com.chaosblade.svc.taskresource.entity.System sys;

  public List<ApiTopologyEdge> topologyEdges;
```
